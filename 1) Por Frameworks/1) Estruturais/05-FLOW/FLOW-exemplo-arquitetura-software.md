# Exemplo FLOW — Proposta de Arquitetura de Software
> Grupo 4 · Arquitetura e Projeto de SW · Perfil: Arquiteto de Soluções

---

⚙️ **Function**
Propor uma arquitetura de software para o cenário descrito, identificando componentes, camadas e padrões de design mais adequados.

📊 **Level**
Nível arquitetural sênior — deve considerar escalabilidade, separação de responsabilidades, padrões de mercado (ex: Clean Architecture, Hexagonal) e aderência ao stack tecnológico da Ayesa.

📤 **Output**
Documento de arquitetura com:
- Diagrama textual da arquitetura proposta (camadas e componentes)
- Justificativa das escolhas arquiteturais
- Tecnologias recomendadas por camada
- Riscos técnicos identificados e mitigações sugeridas

**Contexto:**
- Sistema: portal de autoatendimento para clientes de um projeto SAP
- Stack disponível: .NET Core (backend), React (frontend), Azure (cloud)
- Requisitos não funcionais: alta disponibilidade, autenticação SSO com Azure AD, logs centralizados

🏆 **Win Metric**
A resposta será bem-sucedida se: um desenvolvedor sênior conseguir começar a implementação com base no documento sem precisar de reuniões adicionais de alinhamento, e se os riscos identificados forem específicos o suficiente para virar itens de backlog.
