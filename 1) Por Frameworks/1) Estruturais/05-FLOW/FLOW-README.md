# FLOW — Function, Level, Output, Win Metric

## O que é

O framework **FLOW** define o que o modelo deve fazer, em qual profundidade, como entregar e — o mais importante — **qual é o critério de sucesso**. O *Win Metric* é o diferencial: você define o que significa "tarefa bem-feita" antes de receber a resposta.

| Elemento | Pergunta que responde |
|---|---|
| ⚙️ **Function** | Qual é a função necessária que o modelo precisa executar? |
| 📊 **Level** | Qual o nível de profundidade esperado? (básico / intermediário / sênior / executivo) |
| 📤 **Output** | Qual a saída esperada? (formato, estrutura) |
| 🏆 **Win Metric** | Como sabemos que a tarefa foi bem concluída? Qual o critério de sucesso? |

---

## Quando usar

- Quando você precisa definir **o que é sucesso** antes de receber a resposta
- Tarefas de avaliação, tomada de decisão ou comparação de alternativas
- Quando o **nível de profundidade** varia conforme o público (técnico vs. executivo)
- Situações onde respostas vagas ou incompletas são um risco real

## Diferencial em relação aos outros frameworks

O FLOW é o único que exige um **critério de conclusão** explícito (Win Metric). Enquanto o RISE define o processo e o RAIN define o limite numérico, o FLOW define o que significa "ganhar" — ideal para decisões e avaliações onde a qualidade do raciocínio importa tanto quanto o output.
