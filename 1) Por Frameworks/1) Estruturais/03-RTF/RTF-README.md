# RTF — Role, Task, Format

## O que é

O framework **RTF** é o mais direto e objetivo — define **quem fala**, **o que fazer** e **como entregar**. Ideal quando o formato do output é tão importante quanto o conteúdo em si.

| Elemento | Pergunta que responde |
|---|---|
| 🎭 **Role** | Quem o modelo deve "ser"? Qual o ponto de vista? |
| 📋 **Task** | Qual é a tarefa específica a executar? |
| 📄 **Format** | Como o resultado deve ser entregue? |

---

## Quando usar

- Quando você sabe exatamente o que quer e só precisa definir **como receber**
- Outputs que precisam de um formato padronizado (tabela, checklist, relatório, e-mail)
- Tarefas recorrentes onde o formato é fixo e só o conteúdo muda
- Comunicações formais que precisam de um tom específico

## Diferencial em relação aos outros frameworks

O RTF é o framework mais enxuto — sem contexto adicional, sem Numeric Target, sem próximo passo. Use quando a tarefa é clara e o formato é o ponto crítico. Para tarefas com mais contexto ou que exigem raciocínio em etapas, prefira RAIN ou 4Is.

## Contribuindo

Adicione novos exemplos seguindo o padrão:
- Indique o **contexto** (Grupo + tema) e o **perfil** do usuário
- O *Format* deve ser sempre específico — evite "em formato de texto corrido"
- Prefira formatos que a equipe já usa no dia a dia (tabela, e-mail, checklist, relatório)
