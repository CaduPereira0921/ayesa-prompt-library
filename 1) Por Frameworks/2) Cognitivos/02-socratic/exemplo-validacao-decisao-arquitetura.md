# Exemplo Socratic — Validação de Decisão de Arquitetura
> Grupo 4 · Arquitetura de SW · Perfil: Arquiteto de Soluções / Tech Lead

---

A equipe técnica decidiu usar uma integração ponto a ponto via API REST entre SAP e o sistema de CRM do cliente, sem middleware.

Antes de validar essa decisão, atue como um arquiteto de soluções sênior e questione as premissas dessa escolha.

**Instruções:**
- Formule as perguntas que precisam ser respondidas antes de confirmar essa arquitetura
- Questione: volume de transações, disponibilidade esperada, tratamento de erros, versionamento, monitoramento
- Para cada ponto não respondido, descreva qual risco ele representa no médio prazo
- Conclua com uma recomendação: confirmar a decisão, revisá-la ou escalar para uma decisão de arquitetura formal
