# ROSE — Role, Objective, Steps, Expected Result

## O que é

O framework **ROSE** estrutura o prompt com foco em um **objetivo claro** e um **resultado concreto esperado**. A diferença central é que o ponto de partida é a meta — não o material de análise.

| Elemento | Pergunta que responde |
|---|---|
| 🎭 **Role** | Quem o modelo deve "ser"? Qual o ponto de vista? |
| 🎯 **Objective** | Qual é o objetivo a ser alcançado? |
| 🪜 **Steps** | O que deve ser feito, passo a passo? |
| 📦 **Expected Result** | Qual é o entregável concreto esperado ao final? |

---

## Quando usar

- Quando o **objetivo final** é o ponto de partida, não o material de análise
- Tarefas de criação e construção (proposta, especificação, plano)
- Quando você quer descrever o **entregável** com precisão antes de receber a resposta
- Situações onde o caminho (Steps) precisa ser guiado para garantir completude

## Diferencial em relação ao RISE

| | RISE | ROSE |
|---|---|---|
| Ponto de partida | Material/contexto (Input) | Objetivo a alcançar (Objective) |
| Critério final | Padrão de qualidade (Expectation) | Entregável concreto (Expected Result) |
| Melhor para | Análises e revisões | Criação e construção |

## Contribuindo

Adicione novos exemplos seguindo o padrão:
- Indique o **contexto** (Grupo + tema) e o **perfil** do usuário
- O *Objective* deve ser uma meta clara e verificável
- O *Expected Result* deve descrever o entregável com detalhes suficientes para não gerar dúvidas
