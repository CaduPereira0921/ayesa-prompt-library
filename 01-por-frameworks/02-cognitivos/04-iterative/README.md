# Iterative Prompting

## O que é

O **Iterative Prompting** é uma técnica de refinamento progressivo — em vez de tentar obter a resposta perfeita em um único prompt, você constrói o resultado em camadas, ajustando e aprofundando a cada rodada.

## Quando usar

- Documentos complexos que precisam de múltiplas revisões (propostas, especificações, relatórios)
- Quando o resultado inicial é bom mas precisa ser adaptado para o público ou contexto
- Geração de código que precisa de refinamentos incrementais
- Situações onde você não sabe exatamente o que quer até ver a primeira versão

## Como estruturar

**Rodada 1 — Esboço:** peça uma versão inicial simples e rápida
**Rodada 2 — Refinamento:** ajuste tom, profundidade, estrutura
**Rodada 3 — Adaptação:** adapte para o público-alvo específico
**Rodada 4+ — Detalhamento:** aprofunde seções específicas conforme necessário

## Dica de ouro

Salve o prompt de cada rodada — eles se tornam templates reutilizáveis para situações similares no futuro.

## Diferencial cognitivo

O Iterative Prompting reconhece que **o melhor prompt para uma tarefa complexa raramente é o primeiro**. É uma conversa, não um comando.
