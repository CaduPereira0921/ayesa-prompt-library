# RISE — Role, Input, Steps, Expectation

## O que é

O framework **RISE** estrutura o prompt com foco no **processo de raciocínio** e na **qualidade esperada** do resultado. Vai além de dizer o que fazer — define *como fazer* e *o que é um bom resultado*.

| Elemento | Pergunta que responde |
|---|---|
| 🎭 **Role** | Quem o modelo deve "ser"? Qual o ponto de vista? |
| 📂 **Input** | Qual o material de apoio ou contexto disponível? |
| 🪜 **Steps** | O que deve ser feito, passo a passo? |
| ✅ **Expectation** | Como é um resultado bem-feito? Qual o padrão esperado? |

---

## Quando usar

- Quando a **metodologia de análise** é tão importante quanto o resultado
- Tarefas técnicas que exigem um raciocínio estruturado e verificável
- Quando você quer estabelecer um **padrão de qualidade** claro para o output
- Revisões, avaliações e análises onde o "como" faz diferença

## Diferencial em relação aos outros frameworks

O RISE é o único que explicita tanto os **passos** quanto a **expectativa de qualidade**. Enquanto o RTF define o formato e o RAIN define o limite numérico, o RISE define o processo e o padrão — ideal para tarefas técnicas complexas onde o caminho importa tanto quanto o destino.
