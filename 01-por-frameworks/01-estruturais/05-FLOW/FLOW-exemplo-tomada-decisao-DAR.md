# Exemplo FLOW — Tomada de Decisão (DAR)
> Grupo 2 · Gestão de Projetos · Perfil: Gerente de Projetos / Arquiteto de Soluções

---

⚙️ **Function**
Avaliar alternativas de solução técnica e recomendar a mais adequada para o contexto do projeto.

📊 **Level**
Profundidade técnica sênior — a análise deve ir além do óbvio, considerando custo total de implementação, riscos de integração e sustentabilidade a longo prazo.

📤 **Output**
Uma matriz de decisão comparativa com:
- Critérios de avaliação (ex: custo, prazo, complexidade, risco)
- Pontuação de cada alternativa por critério (escala 1–5)
- Justificativa para cada pontuação
- Recomendação final destacada com argumentação objetiva

**Contexto:**
- Projeto: integração entre SAP S/4HANA e sistema legado de faturamento
- Alternativas em avaliação:
  1. Integração via middleware (SAP Integration Suite)
  2. Integração ponto a ponto via API REST
  3. Replicação de dados via arquivo flat (batch)
- Restrição: budget limitado e prazo de 3 meses para go-live

🏆 **Win Metric**
A resposta será bem-sucedida se: a recomendação for única e clara (sem "depende"), os critérios forem relevantes para o contexto SAP, e qualquer gerente de projeto conseguir justificar a decisão para o cliente com base na matriz.
